Habitus

Mon 27 October 2014 | tags: project-portfolio, research, -- (permalink)

  • Project title: Habitus
  • Code: GitHub Repository

Summary

This project aims to capture data from the user of a computer, using a non-invasive approach. Currently, the Habitus software, written in Python, uses a mix of Pyobjc and AppleScript in order to retrieve data from software usage and visited websites.

The approach taken is the use of the threading Python module, together with timers. A timer is used to retrieve the data in fixed intervals, and threads are used to retrieve and send data.

To prevent the need for server-side scripting, a Python class has been programmed in order to send e-mails with attachments, and the info is sent to my own Dropbox account, where my own data is being stored.

I am currently learning Objective-C in order to change the approach taken in this project. The use of timers brings several limitations, and it is very inefficient. Much data may be lost, and therefore, a purely event-driven approach must be employed. Apparently, this can be done with the use of the old Carbon API.
